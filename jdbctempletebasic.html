JDBC Template Basics

1. JDBC Template Basics

JdbcTemplate is a powerful and flexible tool that simplifies JDBC code. Below are some of the key methods provided by JdbcTemplate, with links to detailed examples:

Methods in JdbcTemplate:

  • execute() - Refer to CRUD Operations
  • update() - Refer to CRUD Operations
  • query() - Refer to CRUD Operations
  • queryForObject() - Refer to CRUD Operations
  • queryForList() - Refer to CRUD Operations
  • batchUpdate() - Refer to Batch Processing

1. execute()

This method is used for executing a SQL statement (DDL or DML) that doesn't return a result set.


String sql = "CREATE TABLE users (id INT, name VARCHAR(50), email VARCHAR(50))";
jdbcTemplate.execute(sql);

2. update()

The update() method is used for executing INSERT, UPDATE, or DELETE statements.


// Insert example
String sql = "INSERT INTO users (id, name, email) VALUES (?, ?, ?)";
jdbcTemplate.update(sql, 1, "John Doe", "john@example.com");

3. query()

This method is used to execute a SELECT query and map the result set to an object.


String sql = "SELECT * FROM users";
List<User> users = jdbcTemplate.query(sql, new UserRowMapper());

4. queryForObject()

The queryForObject() method is used when you expect a single result from a query.


String sql = "SELECT name FROM users WHERE id = ?";
String name = jdbcTemplate.queryForObject(sql, new Object[]{1}, String.class);

5. batchUpdate()

The batchUpdate() method allows executing a batch of SQL statements efficiently.


String sql = "INSERT INTO users (id, name, email) VALUES (?, ?, ?)";
List<Object[]> batchArgs = new ArrayList<>();
batchArgs.add(new Object[]{1, "John", "john@example.com"});
batchArgs.add(new Object[]{2, "Jane", "jane@example.com"});
batchArgs.add(new Object[]{3, "Doe", "doe@example.com"});
jdbcTemplate.batchUpdate(sql, batchArgs);
